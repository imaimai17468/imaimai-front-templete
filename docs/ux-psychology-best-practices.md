## 8つのコアルール

### 1. シンプルに保つ
**統合原則**: 認知負荷、ミラーの法則、オッカムの剃刀、認知的流暢性、プログレッシブディスクロージャー

- ✅ 情報は一度に**5-7項目**まで
- ✅ 段階的に表示（アコーディオン、「もっと見る」）
- ✅ 不要な要素を削除、シンプルな表現
- ✅ 明確な視覚階層、読みやすいフォント
- ❌ 長いリスト、複雑なナビゲーション、専門用語の羅列

**実装例**:
```tsx
// ❌ 悪い例: 一度に全て表示
<div>
  {allFeatures.map(feature => <FeatureCard {...feature} />)}
</div>

// ✅ 良い例: 段階的表示
<div>
  {topFeatures.slice(0, 6).map(feature => <FeatureCard {...feature} />)}
  <Button>さらに見る</Button>
</div>
```

---

### 2. 瞬時にフィードバック
**統合原則**: ドハティの閾値、フィードバックループ、没入感

- ✅ **0.4秒以内**に応答（目標）
- ✅ スケルトンスクリーン、楽観的UI
- ✅ ローディング表示、成功/エラーメッセージ
- ✅ 即座のホバー・アクティブ状態
- ❌ 長時間の無反応、フィードバックなし

**実装例**:
```tsx
// ✅ 楽観的UI + スケルトン
const { mutate } = useMutation({
  onMutate: (newItem) => {
    // 即座にUIを更新
    queryClient.setQueryData(['items'], (old) => [...old, newItem])
  },
  onError: () => {
    // エラー時にロールバック
    queryClient.invalidateQueries(['items'])
  }
})
```

---

### 3. 選択肢を絞る
**統合原則**: ヒックの法則、選択のパラドックス、デフォルト効果、選択アーキテクチャ、パレートの法則

- ✅ 選択肢は**3-5個**に制限
- ✅ **推奨オプション**を明示、デフォルト設定
- ✅ カテゴリ分類で整理
- ✅ 重要な20%の機能に注力
- ❌ 数十個の選択肢、すべて同列表示

**実装例**:
```tsx
// ✅ 3つのプランに絞り、推奨を明示
<PricingCards>
  <PlanCard name="ベーシック" />
  <PlanCard name="プロ" recommended={true} /> {/* 推奨 */}
  <PlanCard name="エンタープライズ" />
</PricingCards>
```

---

### 4. 重要なものを大きく近くに
**統合原則**: フィッツの法則、フォンレストルフ効果、美的ユーザビリティ、視覚階層

- ✅ 主要CTAを**大きく**（タッチターゲット44x44px以上）
- ✅ 頻繁操作要素は**近くに**配置
- ✅ 色・サイズ・形状で**差別化**
- ✅ 美しいデザインが使いやすさの認識を向上
- ❌ 小さすぎるボタン、平凡なCTA、遠い配置

**実装例**:
```tsx
// ✅ 主要CTAを目立たせる
<Button
  size="lg"              // 大きく
  variant="primary"       // 目立つ色
  className="min-h-[44px] min-w-[44px]" // タッチ対応
>
  今すぐ始める
</Button>

// ❌ 悪い例: 小さく地味
<Button size="sm" variant="ghost">登録</Button>
```

---

### 5. 進捗を見せる
**統合原則**: ゴールグラディエント効果、ツァイガルニク効果、ピークエンドの法則、自己効力感

- ✅ プログレスバー、達成率、残りステップ表示
- ✅ 「あと1ステップ」「80%完成」
- ✅ 成功体験の強調、ポジティブな終了画面
- ✅ 小さな成功体験の積み重ね
- ❌ 進捗不明、長すぎる単一ステップ

**実装例**:
```tsx
// ✅ ステップ表示 + 進捗バー
<div>
  <ProgressBar value={currentStep} max={totalSteps} />
  <p>ステップ {currentStep} / {totalSteps}</p>
  <p className="text-sm text-muted-foreground">
    あと{totalSteps - currentStep}ステップで完了！
  </p>
</div>
```

---

### 6. 損失と希少性を強調
**統合原則**: 損失回避、希少性効果、パーキンソンの法則、現状維持バイアス

- ✅ 「**残り3個**」「**本日限定**」「**あと10分**」
- ✅ 損失の具体的可視化
- ✅ カウントダウン、タイムリミット
- ✅ リスク最小化の説明
- ❌ 虚偽の希少性、過度な煽り

**実装例**:
```tsx
// ✅ 希少性と緊急性
<Badge variant="destructive">残り3個</Badge>
<CountdownTimer endTime={limitTime} />
<p className="text-destructive font-medium">
  本日23:59まで限定価格
</p>
```

---

### 7. 社会的証明を活用
**統合原則**: 社会的証明、ハロー効果、返報性の原理、ジェイコブの法則、単純接触効果

- ✅ **実績数**・**評価**・**レビュー**表示
- ✅ 「◯◯人が利用中」「★4.8 (1,234件)」
- ✅ 著名顧客ロゴ、受賞歴、専門家推薦
- ✅ 業界標準UIパターンの採用
- ❌ 証拠のない主張、独自すぎるUI

**実装例**:
```tsx
// ✅ 社会的証明の複合表示
<div className="flex items-center gap-4">
  <div className="flex items-center">
    <Star className="fill-yellow-400" />
    <span className="font-semibold">4.8</span>
    <span className="text-muted-foreground">(1,234件)</span>
  </div>
  <Badge>10,000人以上が利用中</Badge>
</div>
```

---

### 8. 慣習に従う
**統合原則**: ジェイコブの法則、アフォーダンス、テスラーの複雑性保存の法則、ポステルの法則

- ✅ **業界標準パターン**を採用（ロゴ左上、検索右上）
- ✅ ボタンは押せる見た目、リンクは下線・青色
- ✅ 柔軟な入力許容（自動補正、多様な形式）
- ✅ システム側で複雑性を吸収
- ❌ 不必要な独自性、機能不明なデザイン、厳格すぎる入力制限

**実装例**:
```tsx
// ✅ 標準的な配置とアフォーダンス
<header>
  <Logo className="absolute left-4" /> {/* 左上 */}
  <SearchBar className="absolute right-4" /> {/* 右上 */}
</header>

<Button className="rounded-lg shadow-sm"> {/* 押せる見た目 */}
  クリック
</Button>

// ✅ 柔軟な入力
<Input
  type="tel"
  pattern="[0-9-]*"
  placeholder="090-1234-5678"
  // 自動でハイフン挿入・削除
/>
```

## クイックチェックリスト

実装時に確認すべき8項目:

```
[ ] 1. シンプル: 情報は5-7項目、段階的表示、不要削除
[ ] 2. 瞬時: 0.4秒以内応答、スケルトン、即座のフィードバック
[ ] 3. 絞る: 選択肢3-5個、推奨明示、デフォルト設定
[ ] 4. 大きく近く: CTA 44x44px以上、色・サイズで差別化
[ ] 5. 進捗: プログレスバー、残りステップ、達成感演出
[ ] 6. 希少性: 「残り◯個」「本日限定」、カウントダウン
[ ] 7. 社会的証明: 実績数、評価、レビュー、著名ロゴ
[ ] 8. 慣習: 業界標準UI、ボタンは押せる見た目、柔軟入力
```
