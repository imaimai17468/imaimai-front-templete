# 0014. Golden evals govern the review pipeline; delta re-review mode; on-demand repo audit

- Status: accepted
- Date: 2026-07-10

## Context

Two pressures surfaced together (design:
`docs/superpowers/specs/2026-07-10-model-quality-continuity-design.md`):

1. **Model continuity.** The quality *enforcement* stack (deterministic gates
   ADR-0013, pinned review agents ADR-0011, Aegis compilation) is
   model-independent, but the *judgment* of the strongest available model
   (currently Fable) — design decisions, investigation playbooks — dies with
   each session unless it happens to land in an ADR. Nothing measured whether
   a weaker parent (e.g. Opus) plus the existing scaffolding actually holds
   quality, and nothing systematically converts best-model judgment into
   executable artifacts.
2. **Review pipeline waste.** ADR-0013's stamp-clearing forces a re-review
   after every post-review fix. Session measurements (2026-07-10) showed
   re-reviews re-reading the full diff and re-running whole-project
   verification the parent had already run (~40–105k tokens, 1–5 min per
   run).

The cycle of shadcn/improve ("audit with your best model, execute with a
cheap one; plans are portable Markdown") addresses (1), but importing it
as-is would duplicate Aegis (repo-convention mapping, design-doc ingestion)
and introduce a third artifact format beside ADRs and
`docs/superpowers/specs/` plans.

## Decision

1. **Golden evals govern the review pipeline.** Seeded-defect fixtures +
   expected findings live in `docs/superpowers/evals/review-diff/` (patch
   files with self-declared base; run protocol and scoring in its README).
   A model-tier change to `code-reviewer` or `review-verifier`, or any
   load-bearing edit to `review-diff`, requires a scored eval run recorded in
   `results/` —
   measurement replaces judgment calls (superseding ADR-0011's ad-hoc
   empirical-tuning obligation for that skill). `verify-spec` /
   `spec-verifier` keep the ADR-0011 empirical-tuning obligation until a
   verify-spec eval exists — recorded debt, not an implicit waiver.
   *(Resolved 2026-07-12: the verify-spec eval now exists with
   tier-discriminating fixtures sx-01..03 — see
   `docs/superpowers/evals/verify-spec/` — so the spec pipeline is under the
   same scored-eval regime; the first tier comparison kept both spec agents
   on opus.)*
2. **`review-diff` gains a delta re-review mode** (amends the ADR-0011
   procedure; stamp semantics unchanged). A re-dispatch in the same task
   cycle passes the prior report verbatim plus a delta description; the
   reviewer scopes to the delta, skips whole-project verification re-runs,
   keeps the verifier child mandatory, and **fails closed to full mode** on
   any precondition break. The verifier child must be awaited in the
   foreground. Measured effect (2026-07-10): quality 4/4 with 0 FP
   preserved; −62% tool uses and no whole-project verification re-runs on
   the smallest possible diff (the −13% token delta sits within observed
   run-to-run variance; the wall-time comparison against the pre-mandate
   baseline is invalid — see the results file). Savings scale with the
   unchanged portion of the diff. Known trade: delta mode does not
   re-examine previously-clean hunks of the same diff, so a fix that breaks
   reviewed-but-clean code is invisible to it — use full mode after major
   rework.
3. **On-demand `repo-audit` skill** converts best-model judgment into the
   existing rails only: audit lanes cover what gates cannot catch
   (architecture drift, security posture, dependency strategy, docs/DX);
   knowledge outputs become ADR/AGENTS.md edits shipped through aegis-share;
   work outputs become `docs/superpowers/specs/` plans. No third format, not
   scheduled, not CI-run. AGENTS.md gains a "Model continuity (non-Fable
   parent)" section: pinned procedures are not re-derived, design judgment
   escalates to the strongest available model, weaker parents verify more.

## Alternatives considered

- **Adopt shadcn/improve as-is**: rejected — its repo-mapping and doc
  ingestion duplicate Aegis, and its plan layout adds a third artifact
  system to keep consistent.
- **No evals (keep tuning by judgment)**: rejected — model-tier and
  procedure changes stay vibes-based, exactly what made ADR-0011's
  benchmark necessary in the first place.
- **Build-first ordering (ship delta mode, evaluate later)**: rejected —
  ADR-0011's code-graph lesson: features justified by reasoning alone can
  be measured net-negative. The eval was built first and the baseline
  recorded before the mode landed.

## Consequences

- Re-reviews after fixes become cheap enough that ADR-0013's strictness
  stays affordable; the delta fixture keeps this claim testable.
- Eval fixtures rot as the codebase evolves; the run protocol requires
  regenerating or retiring stale fixtures in the same run. The current set
  is saturated (5/5 baseline) — it detects regression, not headroom; add
  harder fixtures before using it to discriminate between strong models.
- `repo-audit` earns its keep or leaves: two consecutive audits with no
  actionable output trigger a deletion proposal (retention rule to be
  encoded in the skill when it is created — decision 3 is accepted but not
  yet implemented at the time of writing).
- Wall-time comparisons against the pre-2026-07-10 baseline are skewed by
  the foreground-wait mandate; the re-run columns in
  `docs/superpowers/evals/review-diff/results/2026-07-10-delta-mode.md`
  are the cost reference.
